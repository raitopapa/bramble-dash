# Bramble's Dash

ブラウザだけで遊べる、レトロ風の横スクロールアクションゲームです。Canvas で描画し、キーボード・タッチ操作、BGM / 効果音、複数ステージ、パワーアップ、敵キャラクター、ステージクリア演出までを `index.html` にまとめています。

## 遊び方

### ローカル起動

このリポジトリはビルド不要です。Node.js 18 以上がある場合は、同梱の静的サーバーを使えます。

```bash
npm start
```

起動後、ブラウザで <http://127.0.0.1:8000/> を開きます。

Node.js を使わない場合は、Python の簡易サーバーでも確認できます。

```bash
python3 -m http.server 8000
```

> `file://` で直接開いても動作する想定ですが、ブラウザの音声・フォント・セキュリティ制限を避けるため、ローカルサーバー経由での確認を推奨します。

### 操作方法

| 操作 | キーボード | タッチ |
| --- | --- | --- |
| 移動 | `←` / `→` または `A` / `D` | 左右ボタン |
| しゃがむ | `↓` または `S` | 下ボタン |
| ジャンプ | `Space` / `↑` / `W` / `Z` / `K` | `A` ボタン |
| ダッシュ / ファイア | `X` / `J` / `Shift` | `B` ボタン |
| ポーズ | `P` / `Enter` | 画面ボタンなし |
| ミュート切替 | `M` | 音符ボタン |
| 全画面 | - | 全画面ボタン |

## 現在の実装

- レスポンシブな Canvas ステージ
- タイトル、プレイ中、ポーズ、死亡、コースクリア、ゲームオーバー、全クリア画面
- 3 ステージ構成
  - `1-1`: オーバーワールド入門ステージ
  - `1-2`: 穴・土管・敵配置を増やしたオーバーワールドステージ
  - `2-1`: 天井と洞窟テーマを持つステージ
- プレイヤーアクション
  - 横移動
  - 可変ジャンプ
  - ダッシュ
  - しゃがみ
  - 踏みつけ
  - パワーアップによる大きい姿 / ファイア姿
- 敵キャラクター
  - `Stomper`: 踏むと倒せる歩行敵
  - `Shellback`: 踏むと甲羅になり、蹴って他の敵を倒せる敵
  - `Chomper`: 土管から出てくる敵
- アイテム / オブジェクト
  - コイン
  - キノコ
  - ファイアフラワー
  - ブロック / レンガ / 階段 / 土管 / ゴールポール
- Web Audio API による BGM と効果音
- タッチデバイス向けの画面内操作ボタン

### 自動チェック

`.github/workflows/smoke-test.yml` で GitHub Actions の smoke test を設定しています。Pull Request、`main` / `master` / `work` への push、手動実行で `npm test` が走るため、最低限の起動確認を GitHub 上でも自動化できます。

### 自動マージ

`.github/workflows/auto-merge.yml` で、`auto-merge` ラベル付き Pull Request を自動マージできます。`Smoke Test` workflow が Pull Request 上で成功すると、対象 PR に `auto-merge` ラベルが付いている場合だけ `gh pr merge --squash --delete-branch` を実行します。

初回だけ GitHub 側で以下を確認してください。

1. Repository settings の Actions permissions で、GitHub Actions に write 権限を許可する
2. `auto-merge` ラベルを作成する
3. 自動マージしたい PR に `auto-merge` ラベルを付ける

手動で再実行したい場合は、GitHub Actions の `Auto Merge` workflow を `workflow_dispatch` で起動し、PR 番号を入力してください。

### 変更時の確認観点

- ブラウザコンソールに JavaScript エラーが出ていないこと
- タイトル画面からゲームを開始できること
- `1-1` から最終ステージまで進行できること
- 死亡後に残機が減り、ステージが再開されること
- コイン、ブロック、敵、パワーアップ、ゴールが期待どおり動くこと
- キーボードとタッチ操作の両方で最低限遊べること
- 画面リサイズやスマホ縦横回転で破綻しないこと
